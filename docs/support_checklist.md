# Support Checklist

## First Response

- Confirm branch, user, device, time, module, and receipt/reference number.
- Check `/system/errors`.
- Check audit logs by user/action/date.
- Check payment reconciliation for non-cash payment disputes.
- Check backup status before any data repair.

## Common Issues

- Login fails: confirm username, status active, role assigned, and branch assigned.
- Mobile money sale blocked: enter reference or transaction ID.
- Missing data: confirm branch filter and user role.
- Stock mismatch: check stock movements, refunds, damage records, and stock count sessions.

## Escalation

Escalate immediately for data loss, failed backup, payment mismatch, unauthorized refund, audit tampering attempt, or production database migration failure.
